## 2. 기능별 상세 명세

### 2.1 이력서 분석

#### 2.1.1 입력
- **파일 업로드**: PDF / DOCX (최대 10MB)
- **텍스트 붙여넣기**: 자유 텍스트 입력

#### 2.1.2 처리 흐름
```
1. 파일 텍스트 추출 (PDF: pdf-parse, DOCX: mammoth)
2. PII 마스킹 (이름·연락처·이메일 → [MASKED])
3. 플랜 확인 → 사용량 체크
4. Claude API 호출 (tool_use 강제)
   - FREE: baseTool (커리어 경로 1개 포함, 단일 call)
   - PRO/EXPERT: proBasicTool (기본 분석) → proCareerTool (커리어 3개) 순차 실행
5. 결과 Supabase 저장 (PRO 이상)
6. 원본 파일 Supabase Storage 보존 (선택)
```

#### 2.1.3 출력 필드
| 필드 | 타입 | 설명 |
|------|------|------|
| job_title | string | 현재/목표 직무명 |
| scores.job_fit | number | 직무 적합도 0~100 |
| scores.market_competitiveness | number | 시장 경쟁력 0~100 |
| scores.growth_potential | number | 성장 가능성 0~100 |
| summary | string | 종합 요약 (2~3문장) |
| strengths | string[] | 핵심 강점 (최대 4개) |
| improvements | string[] | 개선 포인트 (최대 4개) |
| keywords | string[] | 핵심 키워드 (최대 8개) |
| career_paths | CareerPath[] | 커리어 방향 (FREE: 1개, PRO+: 3개) |
| candidate_name | string? | 이름 (마스킹 전 추출) |

#### 2.1.4 CareerPath 구조
```typescript
{
  type: 'BASELINE' | 'RECOMMENDED' | 'STRETCH'
  label: string           // 경로 설명
  title: string           // 직무명
  salary_range: string    // 예: "4,500만원~6,500만원"
  salary_bands: [         // 연봉 밴드 (3개: 1년/3년/5년 뒤)
    { period: string, min: number, max: number }
  ]
  points: string[]        // 실전 조언 2개
}
```

#### 2.1.5 분석 보완 (Refine)
- 사용자가 추가 정보 입력 (최대 500자)
- 기존 분석 기반으로 보완 분석 스트리밍
- 계정당 1회 한도

### 2.2 JD 적합도 분석

#### 2.2.1 입력
- 회사명 (필수)
- 포지션명 (선택)
- 채용공고 텍스트 (필수)
- 분석할 이력서 선택 (기존 저장 분석 결과에서)

#### 2.2.2 처리 흐름
```
1. 플랜/쿠폰 확인
2. Claude API 웹 검색으로 회사 실시간 정보 수집 (최대 2회 검색)
3. 이력서 분석 결과 요약 → JD와 비교 분석
4. Claude API tool_use 호출 (claude-haiku)
5. 결과 Supabase jd_analyses 테이블 저장
```

#### 2.2.3 출력 필드
| 필드 | 설명 |
|------|------|
| fit_score | 적합도 점수 0~100 |
| recommendation | APPLY / CONSIDER / SKIP |
| verdict | 한 줄 판정 |
| company_insight | 회사 특성 요약 (웹 검색 기반) |
| jd_interpretation | JD 숨은 요구역량 해석 |
| matching_points | 매칭 강점 3개 |
| gaps | 부족한 점/리스크 2~3개 |
| pitch_points | 어필 전략 3~4개 |

### 2.3 이력서 자동 생성

#### 2.3.1 입력
- 분석 이력서 선택 (보존된 파일 필요)
- JD 분석 결과 선택 (선택적)
- 생성 모드 선택:
  - **업로드 양식 기반**: 원본 파일 양식 유지
  - **자율 포맷**: AI가 섹션 구성 (텍스트 붙여넣기 이력서)
  - **양식 업로드**: 사용자 제공 DOCX 양식 사용

#### 2.3.2 처리 흐름
```
1. Supabase Storage에서 원본 파일 다운로드
2. 파일 텍스트 추출
3. PII 마스킹 → AI 재작성 → PII 복원
4. DOCX 생성 (docx 라이브러리)
5. Base64 인코딩 → 클라이언트 전달 → 자동 다운로드
```

#### 2.3.3 출력
- `{이름}_이력서_{날짜}.docx` 파일
- 주요 변경사항 목록 (최대 5개)

### 2.4 면접 가이드

#### 2.4.1 입력
- 이력서 분석 결과 선택 (필수)
- JD 분석 결과 선택 (선택)
- 면접 형식 입력 (예: "1차 실무 면접")
- 면접관 정보 입력 (예: "팀장급")
- 추가 메모 (선택)

#### 2.4.2 출력 섹션 (6개)
| 섹션 | 내용 |
|------|------|
| SECTION 1 | 핵심 포지셔닝 메시지 |
| SECTION 2 | 자기소개 설계 |
| SECTION 3 | 예상 질문·답변 가이드 (5종) |
| SECTION 4 | 강점·리스크 분석 |
| SECTION 5 | 역질문 추천 3개 |
| SECTION 6 | 면접 전 체크리스트 |

#### 2.4.3 저장 정책
- DB 저장: 생성일 + 10일 후 자동 만료
- HTML 파일 다운로드 제공

### 2.6 관리자 대시보드

- 등록자 이메일 클릭 → 유저 상세 모달:
  - 플랜·사용량
  - 등록한 쿠폰 목록
  - 이력서 분석 내역 (최근 5건)
  - JD 분석 내역 (최근 5건)

## 3. 사용자 흐름 (User Flow)

### 3.1 신규 가입 흐름
```
랜딩 페이지 → [무료로 시작하기] → Google 로그인
→ after-login 처리 (users 테이블 생성) → /analyze 리다이렉트
```

### 3.2 이력서 분석 흐름
```
/analyze → 파일업로드 or 텍스트입력 → 동의체크 → [분석 시작하기]
→ 로딩 (7초 간격 진행 메시지) → 결과 표시
→ (PRO+) 커리어 방향 자동 로드 → 저장 완료 알림
```

### 3.3 JD 분석 흐름
```
JD기반 분석 탭 → 이력서 선택 → 회사명/포지션/JD 입력
→ [적합도 분석하기] → 로딩 (8초 간격 진행 메시지)
→ 결과 표시 (적합도 점수 + 판정 + 분석)
```

### 3.4 이력서 생성 흐름
```
이력서 생성 탭 → 이력서 선택 → 포맷 선택 모달
→ (양식업로드 선택 시) 파일 업로드 모달
→ JD 선택 모달 → 생성 중... → .docx 자동 다운로드
```
