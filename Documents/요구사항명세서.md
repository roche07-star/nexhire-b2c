**프로젝트명:** Jobizic (코드명: adam)  
**버전:** v1.0  
**작성일:** 2026-05-31  
**작성자:** HEDING

## 1. 개요

### 1.1 목적
한국 구직자를 대상으로 AI 기반 이력서 분석 및 커리어 전략 수립을 지원하는 B2C SaaS 플랫폼의 요구사항을 정의한다.

### 1.2 범위
- 이력서 AI 분석 및 점수화
- JD(채용공고) 적합도 분석
- AI 기반 이력서 자동 생성
- 면접 가이드 생성
- 커리어 방향 분석
- 플랜 기반 사용량 관리 및 쿠폰 시스템
- 관리자 대시보드

### 1.3 용어 정의
| 용어 | 정의 |
|------|------|
| JD | Job Description, 채용공고 |
| PII | Personally Identifiable Information, 개인식별정보 |
| DOCX | Microsoft Word 문서 포맷 |
| 플랜 | 서비스 이용 등급 (FREE / PRO / EXPERT) |
| 쿠폰 | 특정 기능의 추가 사용권 |

## 2. 비기능 요구사항

### 2.1 성능
- **FR-NF-001** API 응답 시간: 이력서 분석 90초 이내, JD 분석 60초 이내
- **FR-NF-002** 동시 사용자: Vercel Serverless 기반으로 자동 확장
- **FR-NF-003** 파일 업로드: 최대 10MB, PDF/DOCX 지원

### 2.2 보안
- **FR-NF-004** 인증: Google OAuth 2.0 기반 소셜 로그인만 지원
- **FR-NF-005** PII 마스킹: 이력서 분석 전 이름·연락처·이메일 자동 마스킹 후 AI 전달
- **FR-NF-006** 데이터 격리: 사용자별 분석 결과 접근 제한 (이메일 기준)

### 2.3 가용성
- **FR-NF-007** 서비스 가용성: Vercel + Supabase 기반 99% 이상
- **FR-NF-008** 분석 결과 보존: PRO/EXPERT 사용자 분석 결과 영구 보존, 면접 가이드 10일 보존

### 2.4 운영
- **FR-NF-009** 배포: GitHub main 브랜치 push → Vercel 자동 배포 (CI/CD)
- **FR-NF-010** 모니터링: Vercel 로그 기반 에러 추적

## 4. 제약 조건
- Vercel Serverless Function 최대 실행 시간: 이력서 분석 90초, 기타 60초
- Claude API: claude-haiku-4-5-20251001 기본 사용 (비용/속도 최적화)
- 인증 방식: Google OAuth만 지원 (이메일/비밀번호 미지원)
- 파일 저장: Supabase Storage (resumes 버킷)
