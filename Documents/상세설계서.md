## 2. 데이터베이스 설계

### 2.1 테이블 구조

#### users
```sql
CREATE TABLE users (
  email              TEXT PRIMARY KEY,
  name               TEXT,
  image              TEXT,
  plan               TEXT DEFAULT 'FREE',        -- FREE|PRO|EXPERT
  role               TEXT DEFAULT 'USER',        -- USER|MANAGER
  analyze_count      INTEGER DEFAULT 0,
  jd_count           INTEGER DEFAULT 0,
  rewrite_count      INTEGER DEFAULT 0,
  interview_count    INTEGER DEFAULT 0,
  monthly_reset_at   TIMESTAMPTZ,               -- 플랜 가입일 (초기화 기준)
  created_at         TIMESTAMPTZ DEFAULT now()
);
```

#### analyses
```sql
CREATE TABLE analyses (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_email  TEXT REFERENCES users(email),
  result      JSONB,                             -- 분석 결과 전체
  created_at  TIMESTAMPTZ DEFAULT now(),
  expires_at  TIMESTAMPTZ                        -- FREE: 단기 만료
);

-- result JSONB 구조
{
  job_title: string,
  scores: { job_fit: number, market_competitiveness: number, growth_potential: number },
  summary: string,
  strengths: string[],
  improvements: string[],
  keywords: string[],
  career_paths: CareerPath[],
  plan: string,
  candidate_name?: string,
  _file_path?: string,        -- Supabase Storage 경로 (보존 시)
  refined?: boolean,
  refinement_text?: string
}
```

#### jd_analyses
```sql
CREATE TABLE jd_analyses (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_email  TEXT REFERENCES users(email),
  result      JSONB,
  created_at  TIMESTAMPTZ DEFAULT now(),
  expires_at  TIMESTAMPTZ DEFAULT (now() + INTERVAL '10 years')
);

-- result JSONB 구조
{
  company: string,
  position?: string,
  fit_score: number,
  recommendation: 'APPLY'|'CONSIDER'|'SKIP',
  verdict: string,
  company_insight: string,
  jd_interpretation: string,
  matching_points: string[],
  gaps: string[],
  pitch_points: string[],
  resume_job_title?: string,
  resume_analyzed_at?: string
}
```

#### interview_guides
```sql
CREATE TABLE interview_guides (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_email  TEXT REFERENCES users(email),
  result      JSONB,
  created_at  TIMESTAMPTZ DEFAULT now(),
  expires_at  TIMESTAMPTZ DEFAULT (now() + INTERVAL '10 days')
);
```

#### coupons
```sql
CREATE TABLE coupons (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code        TEXT UNIQUE NOT NULL,             -- NH-{feature}-{6자리}
  feature     TEXT NOT NULL,                    -- resume|jd|rewrite|interview
  price       INTEGER DEFAULT 0,
  issued_to   TEXT,                             -- 지정 이메일 (null이면 누구나)
  claimed_by  TEXT REFERENCES users(email),
  claimed_at  TIMESTAMPTZ,
  used_at     TIMESTAMPTZ,
  expires_at  TIMESTAMPTZ,
  created_at  TIMESTAMPTZ DEFAULT now()
);
```

## 3. API 라우트 설계

### 3.1 분석 API

#### POST /api/analyze
```
설명: 이력서 분석
인증: 필수 (세션)
Content-Type: multipart/form-data
요청:
  - resume: File (PDF/DOCX, optional)
  - resumeText: string (optional, 텍스트 붙여넣기)
  - preserveMode: 'auto'|'replace'|'add'|'skip'
응답: AnalysisResult + { _id, _rewrite_saved, _file_path? }
maxDuration: 90초
```

#### GET /api/analyze/list
```
설명: 저장된 분석 목록 (최근 20개)
인증: 필수
응답: { analyses: AnalysisListItem[] }
```

#### POST /api/analyze/[id]/expand
```
설명: 커리어 경로 확장 (PRO/EXPERT, 실패 시 재시도)
인증: 필수
응답: { career_paths: CareerPath[] }
maxDuration: 45초
```

#### POST /api/analyze/refine
```
설명: 분석 보완 (스트리밍)
인증: 필수
요청: { analysisId, userInput }
응답: text/plain (stream)
```

#### POST /api/analyze/jd
```
설명: JD 적합도 분석
인증: 필수
요청: { company, position?, jd, analysisResult }
응답: JDAnalysisResult
maxDuration: 60초
```

#### POST /api/analyze/rewrite
```
설명: 이력서 자동 생성
인증: 필수 (EXPERT)
요청: FormData { analysisId, jdAnalysisId?, formatMode, templateFile? }
응답: { docx: base64, filename, changes[] }
maxDuration: 60초
```

#### POST /api/analyze/interview
```
설명: 면접 가이드 생성
인증: 필수 (EXPERT)
요청: { analysisId, jdAnalysisId?, format, interviewer, notes }
응답: InterviewGuideResult
maxDuration: 60초
```

{

## 6. 보안 설계

### 6.1 인증 흐름
```
Google OAuth → NextAuth 세션 생성 → JWT 쿠키
→ API 라우트: auth() 호출 → session.user.email 확인
→ MANAGER 확인: (session.user as { role }).role === 'MANAGER'
```

### 6.2 PII 마스킹 (lib/maskPII.ts)
```
마스킹 대상:
- 이름: (이름|성명|Name) 패턴 → [MASKED_NAME]
- 연락처: 010-xxxx-xxxx 패턴 → [MASKED_PHONE]
- 이메일: xxx@xxx.xxx 패턴 → [MASKED_EMAIL]
- 주민번호: → [MASKED_ID]

복원: 리라이팅 시 마스킹 값 → 원본값 치환
```

### 6.3 데이터 접근 제어
- 모든 DB 쿼리에 `user_email = session.user.email` 조건 필수
- Supabase RLS 미사용 (service role key로 서버에서만 접근)
- Storage URL: 서명된 URL(signed URL) 방식 사용

## 10. 에러 처리 설계

### 10.1 API 에러 코드
| HTTP | 상황 | 메시지 |
|------|------|-------|
| 401 | 비로그인 | 로그인이 필요합니다 |
| 403 | 플랜 부족 | {기능}은 {플랜} 이상에서 이용 가능 |
| 403 | 횟수 초과 | 이번 달 {N}회를 모두 사용했습니다 |
| 400 | 입력 오류 | 필수 입력값 안내 |
| 422 | 파일 파싱 실패 | 파일을 읽을 수 없습니다 |
| 500 | Claude 불완전 응답 | 분석 결과가 불완전합니다. 다시 시도해 주세요 |
| 500 | DB 저장 실패 | 분석 결과 저장 실패 |

### 10.2 클라이언트 에러 처리
- 로딩 중 진행 메시지 표시 (7~8초 간격 순환)
- 에러 시 `다시 시도` 버튼 제공
- 커리어 경로 로딩 실패 시 expand 엔드포인트로 재시도 버튼
