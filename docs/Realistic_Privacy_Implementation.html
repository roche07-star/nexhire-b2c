핵심 메시지

완벽할 필요는 없습니다. 하지만 최소한은 지켜야 합니다.
이 문서는 "법적으로 반드시 해야 하는 최소한"과 "나중에 해도 되는 것"을 구분합니다.
대부분의 스타트업도 이렇게 시작했습니다. 성장하면서 점진적으로 개선하면 됩니다.

1. 우선순위 구분

🚨 Level 1: 법적 필수 (지금 당장)

개인정보보호법 위반 → 과태료/형사처벌 가능성 있음

  • 1️⃣ 개인정보처리방침 작성·게시 - 법적 필수
  • 2️⃣ 개인정보 수집·이용 동의 - 회원가입 시
  • 3️⃣ Supabase RLS 기본 정책 - 사용자별 데이터 분리
  • 4️⃣ HTTPS (TLS) 적용 - 전송 암호화 (이미 적용됨)

⚠️ Level 2: 중요 (1~3개월 이내)

서비스 신뢰도 및 보안 강화

  • ▪ 접근 로그 기록 (감사 추적)
  • ▪ 마이페이지 개인정보 관리 (열람·수정·삭제·다운로드)
  • ▪ 파일 Signed URL (1시간 만료)
  • ▪ Eve 위탁 계약서 (써치펌 이용 시)

✅ Level 3: 권장 (6개월~1년)

성장하면서 점진적으로 개선

  • ▪ 민감정보 암호화 (전화번호, 주소)
  • ▪ 제3자 제공 동의 시스템 (헤드헌터 열람)
  • ▪ 정기 백업 (암호화)
  • ▪ 개인정보 영향평가

📋 Level 4: 선택 (시리즈 A 이후)

대규모 투자 유치 또는 엔터프라이즈 고객 확보 시

  • ▪ ISMS-P 인증
  • ▪ SOC 2, ISO 27001 인증
  • ▪ DLP (워터마크, 마스킹)
  • ▪ IP Whitelist

2. 최소 필수 구현 (이번 주 안에)

2.1 개인정보처리방침 (30분)

간단한 템플릿 사용 → 나중에 법무팀 검토

✅ 할 일

  1. 코난 보고서의 "개인정보처리방침 템플릿" 복사
  2. 회사명, 연락처만 수정
  3. /privacy 페이지에 게시
  4. 푸터에 링크 추가
// app/privacy/page.tsx
export default function PrivacyPage() {
  return (
    <div>
      <h1>개인정보처리방침</h1>
      {/* 템플릿 내용 붙여넣기 */}
    </div>
  )
}

⏱️ 예상 시간: 30분
💰 비용: ₩0 (나중에 법무 검토 시 약 50~100만원)

2.2 개인정보 수집·이용 동의 (1시간)

✅ 할 일

회원가입 시 체크박스 추가

⏱️ 예상 시간: 1시간
💰 비용: ₩0

2.3 Supabase RLS 기본 정책 (2시간)

✅ 할 일

가장 중요! 사용자별 데이터 분리

-- Supabase Dashboard → SQL Editor

-- analyses 테이블
ALTER TABLE analyses ENABLE ROW LEVEL SECURITY;

CREATE POLICY "사용자는 본인 데이터만 조회"
ON analyses FOR SELECT
USING (auth.jwt() ->> 'email' = user_email);

CREATE POLICY "사용자는 본인 데이터만 삽입"
ON analyses FOR INSERT
WITH CHECK (auth.jwt() ->> 'email' = user_email);

CREATE POLICY "사용자는 본인 데이터만 수정"
ON analyses FOR UPDATE
USING (auth.jwt() ->> 'email' = user_email);

CREATE POLICY "사용자는 본인 데이터만 삭제"
ON analyses FOR DELETE
USING (auth.jwt() ->> 'email' = user_email);

-- 다른 테이블도 동일하게 적용

⏱️ 예상 시간: 2시간 (테이블마다 적용)
💰 비용: ₩0

3. 현실적인 타임라인

이번 주 (필수) - 3.5시간

• 개인정보처리방침 게시 (30분)
• 회원가입 동의 체크박스 (1시간)
• Supabase RLS 정책 (2시간)

1개월 이내 (중요) - PRO 플랜과 병행

• 접근 로그 기록
• 파일 Signed URL
• 마이페이지 개인정보 관리

3개월 이내 (권장)

• 민감정보 암호화
• Eve 위탁 계약서 (써치펌 이용 시)
• 정기 백업

1년 이내 (선택)

• 개인정보 영향평가
• 법무팀 정식 검토
• DLP, 워터마크

시리즈 A 이후 (선택)

• ISMS-P 인증
• SOC 2, ISO 27001
• 전담 보안팀 구성

4. 코난 보고서 vs 현실적 방안

❌ 코난 보고서 (이상적)

  • ▪ AES-256 암호화
  • ▪ SOC 2, ISO 27001 인증
  • ▪ CPO (개인정보보호책임자) 전담
  • ▪ 연 1회 개인정보 영향평가
  • ▪ DLP, 워터마크
  • ▪ IP Whitelist
  • ▪ 침해사고 대응팀

💸 예상 비용: 연 5,000만원+
⏱️ 구축 기간: 6개월~1년

✅ 현실적 방안 (최소 필수)

  • ▪ 개인정보처리방침 게시
  • ▪ 수집·이용 동의
  • ▪ Supabase RLS 정책
  • ▪ HTTPS (이미 적용)
  • ▪ 접근 로그 (간단)
  • ▪ 파일 Signed URL

💸 예상 비용: ₩0 (개발 시간만)
⏱️ 구축 기간: 1주일

5. 다른 스타트업도 이렇게 시작했습니다
