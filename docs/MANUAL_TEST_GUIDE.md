# 🧪 수동 테스트 가이드 — 테스 작성

## 🎯 즉시 실행 가능한 테스트 (개발자 도구 활용)

### ✅ 준비사항

**1. Chrome 개발자 도구 활용법**
```
F12 → Network 탭 열기
Filter: Fetch/XHR
Preserve log 체크 (페이지 이동 시 로그 유지)
```

**2. 테스트 계정 준비**
```
헤드헌터 계정: (MANAGER_EMAILS에 등록된 이메일)
일반 사용자 계정: (일반 이메일)
```

**3. 테스트 이력서 파일**
```
tests/fixtures/ 폴더에 저장:
- resume-full.pdf (경력/학력/연봉 모두 있음)
- resume-career-only.pdf (경력만)
- resume-entry.pdf (신입, 정보 없음)
```

## 🚨 테스트 1: 무한루프 확인 (최우선!)

### 방법 1: Network 오프라인 테스트

**목표:** API 실패 시 무한루프 발생 여부 확인

**순서:**
1. Chrome DevTools → Network 탭 열기
2. **"Offline" 체크박스 활성화** (네트워크 차단)
3. 이력서 업로드 → 분석 완료
4. JD 분석 → 완료
5. Network 탭에서 `/api/generate-proposal` 요청 개수 확인

**✅ 성공 기준:**
- API 요청이 **1~3회만** 발생
- 3회 후 멈춤 (Circuit Breaker 작동)
- 에러 메시지: "제안서 생성에 실패했습니다. (1/3)" 또는 "(2/3)" 또는 "(3/3)"

**❌ 실패 (즉시 보고!):**
- API 요청이 10회 이상
- 계속 증가함
- 브라우저가 느려짐

**스크린샷:**
![Network 탭에서 요청 횟수 확인]

---

### 방법 2: 콘솔 로그 모니터링

**순서:**
1. Chrome DevTools → Console 탭 열기
2. Filter: `[제안서 생성]` 또는 `proposal`
3. 이력서 분석 → JD 분석
4. 콘솔 로그 확인

**✅ 성공 기준:**
```
[제안서 생성] 시작: { analysisId: "...", jdId: "...", timestamp: "..." }
[제안서 생성] 성공: { duration: "1234ms" }
```

또는 실패 시:
```
[제안서 생성] 실패: Error 객체
[제안서 생성] 실패: Error 객체
[제안서 생성] 실패: Error 객체
[Circuit Breaker] 5분 후 재시도 가능
```

**❌ 실패 (즉시 보고!):**
```
[제안서 생성] 시작: ...
[제안서 생성] 실패: ...
[제안서 생성] 시작: ...  ← 즉시 재시도!
[제안서 생성] 실패: ...
[제안서 생성] 시작: ...  ← 무한 반복!
```

## 📄 테스트 2: 제안서 경력/학력/연봉 확인

### 방법: 실제 제안서 다운로드 및 확인

**순서:**
1. 이력서 업로드 (경력/학력/연봉 포함)
   ```
   예시 이력서 내용:
   - 이름: 김개발
   - 경력: 네이버 2018~현재 (8년), 카카오 2016~2018 (2년)
   - 학력: 서울대학교 컴퓨터공학과 학사 졸업
   - 연봉: 현재 연봉 8,500만원
   ```

2. JD 분석
   ```
   회사: 토스
   포지션: 백엔드 개발자
   ```

3. 제안서 생성 버튼 클릭 (헤드헌터만 보임)

4. 제안서 다운로드

5. **HTML 파일 열어서 "후보자 기본정보" 섹션 확인**

**✅ 성공 기준:**
```html
<div class="info-row">
  <span class="label">경력</span>
  <span class="value">총 10년 0개월</span>  ← "미기재" 아님!
</div>
<div class="info-row">
  <span class="label">학력</span>
  <span class="value">서울대학교 ... 학사 졸업</span>  ← "미기재" 아님!
</div>
<div class="info-row">
  <span class="label">연봉</span>
  <span class="value">연 8,500만원</span>  ← "미기재" 아님!
</div>
```

**❌ 실패 (즉시 보고!):**
```html
<span class="value">미기재</span>  ← 이것만 나오면 버그!
```

**테스트 변형:**
- 경력만 있고 학력/연봉 없는 경우
- 학력만 있고 경력/연봉 없는 경우
- 모두 없는 경우 (신입)

## 📊 테스트 3: 토큰 사용량 로그 확인

### 방법: Vercel 로그 확인

**순서:**
1. Vercel 대시보드 → 프로젝트 → Functions → Logs
2. 이력서 분석 1건 수행
3. 로그 검색: `[analyze]`

**✅ 성공 기준 (ENABLE_VALIDATION=true, 기본값):**
```
[analyze] 🔍 검증 단계 실행 (ENABLE_VALIDATION=true)
[analyze] 기본 분석 토큰: { input: 2000, output: 500, total: 2500 }
[analyze] 검증 단계 토큰: { input: 1500, output: 300, total: 1800 }
[analyze] 커리어 경로 토큰: { input: 2000, output: 400, total: 2400 }
```

**✅ 성공 기준 (ENABLE_VALIDATION=false, 절약 모드):**
```
[analyze] ⚡ 검증 단계 건너뜀 (토큰 절약 모드)
[analyze] 기본 분석 토큰: { input: 2000, output: 500, total: 2500 }
[analyze] 커리어 경로 토큰: { input: 2000, output: 400, total: 2400 }

← "검증 단계 토큰" 로그 없음! (1,800 tokens 절약)
```

**총합 계산:**
- 검증 활성화: 2,500 + 1,800 + 2,400 = **6,700 tokens**
- 검증 비활성화: 2,500 + 2,400 = **4,900 tokens**
- **절감: 1,800 tokens (26.8%)**

**생성 완료 후:**
- 버튼 텍스트: "📄 후보자 제안서 다운로드"
- 버튼 활성화

## 🎯 테스트 5: Circuit Breaker 쿨다운 시간

**목표:** 3회 실패 후 5분간 차단 확인

**순서:**
1. API를 일부러 실패하게 만들기 (DevTools Network 오프라인)
2. 제안서 생성 3회 시도 → 모두 실패
3. 즉시 4번째 시도
4. 알림 메시지 확인

**✅ 성공 기준:**
```
알림: "제안서 생성이 일시 차단되었습니다. 5분 후 다시 시도해주세요."
```

**5분 후:**
- 다시 시도 가능
- Circuit Breaker 카운터 리셋
