3.2 기술적 보안 조치

// 1. Supabase RLS (Row Level Security) - 사용자별 데이터 분리
CREATE POLICY "사용자는 본인 이력서만 조회"
ON analyses FOR SELECT
USING (auth.jwt() ->> 'email' = user_email);

CREATE POLICY "사용자는 본인 이력서만 수정"
ON analyses FOR UPDATE
USING (auth.jwt() ->> 'email' = user_email);

CREATE POLICY "사용자는 본인 이력서만 삭제"
ON analyses FOR DELETE
USING (auth.jwt() ->> 'email' = user_email);

// 2. 민감정보 암호화 (Supabase Vault 사용)
-- 전화번호, 주소 등은 암호화 저장
INSERT INTO vault.secrets (name, secret)
VALUES ('user_phone_' || user_id, encrypt_phone(phone_number));

// 3. 파일 암호화 (Supabase Storage)
-- PDF/DOCX 파일은 암호화되어 저장
-- Signed URL로만 접근 가능 (1시간 만료)
const { data } = await supabase.storage
  .from('resumes')
  .createSignedUrl(`${userEmail}/${fileName}`, 3600)

// 4. 접근 로그 기록
CREATE TABLE access_logs (
  id UUID PRIMARY KEY,
  user_email TEXT,
  action TEXT,  -- 'view', 'edit', 'delete', 'download'
  resource_id UUID,
  ip_address TEXT,
  user_agent TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

-- RLS: 관리자만 조회
CREATE POLICY "관리자만 접근 로그 조회"
ON access_logs FOR SELECT
USING (auth.jwt() ->> 'role' = 'admin');

3.3 개인정보 처리 흐름

1️⃣ 수집 (Collection)

• 회원가입 시 개인정보처리방침 동의
• 이력서 업로드 시 분석 동의
• 최소 필요 정보만 수집

2️⃣ 저장 (Storage)

• Supabase PostgreSQL (암호화 저장)
• Supabase Storage (파일 암호화)
• RLS 정책으로 접근 제어

3️⃣ 이용 (Use)

• AI 분석 (Claude API - 데이터 미저장 보장)
• 이력서 생성
• 사용자 본인만 조회 가능

4️⃣ 제공 (Sharing)

• 헤드헌터가 구직자 이력서 열람 시 별도 동의 필요
• 열람 사실 구직자에게 알림
• 제3자 제공 기록 보관

5️⃣ 파기 (Deletion)

• 회원 탈퇴 시 즉시 삭제
• 법적 보관 의무 제외 (결제 기록: 5년)
• 백업에서도 완전 삭제 (30일 이내)

3.4 사용자 권리 보장

권리 제공 방법 구현 위치

🔐 추가 보안 레이어

// 1. 팀별 데이터 격리
CREATE POLICY "팀원은 본인 팀 후보자만 조회"
ON eve_candidates FOR SELECT
USING (
  team_id IN (
    SELECT team_id FROM team_members
    WHERE user_email = auth.jwt() ->> 'email'
  )
);

// 2. 역할 기반 접근 제어 (RBAC)
CREATE TABLE team_roles (
  id UUID PRIMARY KEY,
  team_id UUID,
  user_email TEXT,
  role TEXT CHECK (role IN ('admin', 'recruiter', 'viewer')),
  permissions JSONB
);

-- admin: 모든 권한
-- recruiter: 후보자 조회·수정, 파이프라인 관리
-- viewer: 후보자 조회만

// 3. 워터마크 (Watermark)
-- 이력서 다운로드 시 다운로드한 사람 정보 워터마크 삽입
function addWatermark(pdf, userEmail, timestamp) {
  return pdf.addText({
    text: `Downloaded by ${userEmail} at ${timestamp}`,
    opacity: 0.1,
    position: 'bottom-right'
  });
}

// 4. DLP (Data Loss Prevention)
-- 민감정보 마스킹
function maskPhoneNumber(phone) {
  return phone.replace(/(\d{3})-(\d{4})-(\d{4})/, '$1-****-$3');
}

function maskEmail(email) {
  const [local, domain] = email.split('@');
  return local.slice(0, 2) + '***@' + domain;
}

// 5. 접근 제한 (IP Whitelist)
-- 써치펌 사무실 IP만 접근 허용 (옵션)
CREATE TABLE ip_whitelist (
  team_id UUID,
  ip_address CIDR,
  description TEXT
);

4.3 써치펌 계약서 필수 조항

📄 개인정보 처리 위탁 계약서 (예시)

제1조 (위탁 업무의 내용)

수탁자는 위탁자로부터 제공받은 개인정보를 후보자 관리, 파이프라인 추적, 채용 프로세스 진행 목적으로만 처리한다.

제2조 (재위탁 제한)

수탁자는 위탁자의 사전 서면 동의 없이 제3자에게 재위탁할 수 없다.

제3조 (안전성 확보 조치)

수탁자는 개인정보 암호화, 접근권한 관리, 접속기록 보관 등 안전조치를 이행한다.

제4조 (개인정보의 파기)

수탁자는 위탁 계약 종료 시 즉시 개인정보를 파기하고, 파기 증명서를 제출한다.

제5조 (손해배상)

수탁자의 귀책사유로 개인정보 유출 시, 수탁자는 모든 손해를 배상한다.

6. 개인정보처리방침 템플릿

Jobizic 개인정보처리방침

주식회사 ○○○(이하 "회사")는 정보주체의 자유와 권리 보호를 위해 「개인정보 보호법」 및 관계 법령이 정한 바를 준수하여, 적법하게 개인정보를 처리하고 안전하게 관리하고 있습니다.

1. 개인정보의 처리 목적

회사는 다음의 목적을 위하여 개인정보를 처리합니다.
• 회원 가입 및 관리
• AI 이력서 분석 서비스 제공
• JD 적합도 분석 서비스 제공
• 이력서 생성 서비스 제공
• 유료 서비스 결제 및 정산
• 고객 문의 대응 및 서비스 개선

2. 개인정보의 처리 및 보유 기간

• 회원 정보: 회원 탈퇴 시까지
• 이력서 정보: 회원 탈퇴 시까지
• 결제 정보: 5년 (전자상거래법)
• 접속 로그: 3개월 (통신비밀보호법)
