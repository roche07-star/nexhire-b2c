# 🧪 긴급 테스트 체크리스트 — 테스 작성

## ✅ Phase 1: 제안서 경력/학력/연봉 테스트 (15분)

### 테스트 케이스 1: 정상 케이스
**목표:** 경력/학력/연봉이 모두 있는 이력서

**준비:**
```
테스트 이력서 내용:
- 이름: 김테스트
- 경력: 2018년 ~ 현재 (약 8년)
- 학력: 서울대학교 컴퓨터공학과 학사 졸업
- 연봉: 현재 연봉 7,500만원
```

**실행 순서:**
1. [ ] 이력서 업로드
2. [ ] JD 분석 (회사: 네이버, 포지션: 백엔드 개발자)
3. [ ] 제안서 생성 버튼 클릭
4. [ ] 제안서 다운로드
5. [ ] HTML 파일 열어서 확인

**검증:**
- [ ] 경력: "총 8년 0개월" 또는 유사한 형식
- [ ] 학력: "서울대학교 ... 학사 졸업"
- [ ] 연봉: "7,500만원" 또는 "연 7,500만원"
- [ ] "미기재" 없음

**결과:**
- 통과 여부: [ ]
- 실제 표시된 값:
  - 경력: _______________
  - 학력: _______________
  - 연봉: _______________

## ✅ Phase 3: 토큰 사용량 모니터링 (10분)

### 테스트 케이스 6: 검증 비활성화 효과 확인

**준비:**
```bash
# Vercel 환경 변수 설정
ENABLE_VALIDATION=false
```

**실행:**
1. [ ] 이력서 분석 1건
2. [ ] Vercel 로그 확인

**검증:**
```
[analyze] 🔍 검증 단계 실행 (ENABLE_VALIDATION=true)  ← 기본값
또는
[analyze] ⚡ 검증 단계 건너뜀 (토큰 절약 모드)  ← false일 때
```

**로그 확인:**
```
[analyze] 기본 분석 토큰: { input: 2000, output: 500, total: 2500 }
[analyze] 검증 단계 토큰: { input: 1500, output: 300, total: 1800 }  ← 이게 사라지면 성공
[analyze] 커리어 경로 토큰: { input: 2000, output: 400, total: 2400 }
```
